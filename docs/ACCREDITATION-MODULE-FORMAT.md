# Accreditation-Ready Pharmacy Technician Module Format

> **Source:** Provided by the client (2026-06-13) as the target format for building
> polished, accreditation-ready LMS modules for **beginner pharmacy technician**
> learners. Preserves traceability, assessment alignment, and reviewer documentation.
> Aligns to 2026 ASHP/ACPE standards (ongoing formative + summative assessment of
> competencies across didactic, simulation, and experiential modalities).

---

## 1. Module Identification

| Field | Required Content |
|-------|------------------|
| Program Name | Pharmacy Technician Training Program |
| Module Title | Clear title, e.g., "Reading a Prescription Accurately" |
| Module Code/Number | Example: PTM-104 |
| Course Level | Beginner |
| Delivery Mode | Online didactic, simulation, experiential, or blended |
| Estimated Seat Time | Time assigned to module completion |
| Prerequisites | Prior modules or required knowledge |
| Version/Review Date | Version number and next review date |
| Author/Reviewer | SME, instructional designer, pharmacist reviewer, accreditation reviewer |

## 2. Accreditation / Curriculum Crosswalk

Use a crosswalk table for every module. The model curriculum is organized around
goals, objectives, instructional objectives, and sample didactic, simulation, and
experiential activities.

| Accreditation Standard / Curriculum Goal | Module Learning Objective | Instructional Content | Learning Activity | Assessment Method | Evidence in LMS |
|---|---|---|---|---|---|
| Insert applicable standard or goal | Learner will be able to… | Lesson section, reading, table, video, case | Didactic, simulation, discussion, assignment | Quiz, rubric, checklist, scenario, competency validation | Quiz record, completion report, rubric, uploaded assignment |

This table matters because accreditation reviewers want to see that each module is
intentionally mapped to program outcomes, not created as isolated content.

## 3. Module Overview

A short, professional introduction explaining:
- Why the topic matters to pharmacy technician practice.
- How the module fits within the larger Pharmacy Technician curriculum.
- What the learner will be expected to do by the end of the module.
- Whether the module is knowledge-based, skill-based, or competency-based.

**Example:**
> This module introduces beginner pharmacy technician learners to the foundational
> knowledge and practical skills required to read a prescription accurately. The
> module supports safe prescription processing by helping learners identify required
> prescription information, recognize incomplete or unclear information, and understand
> the importance of accuracy before a prescription proceeds through the pharmacy workflow.

## 4. Learning Objectives

Measurable and action-based. Avoid "understand" unless tied to a measurable assessment.
Each objective should be **linked to an assessment**.

| Objective Number | Learning Objective |
|---|---|
| LO1 | Identify the major components of a prescription. |
| LO2 | Distinguish between complete, incomplete, and unclear prescription information. |
| LO3 | Apply a structured review process when reading prescription information. |
| LO4 | Demonstrate accuracy when interpreting prescription details in practice scenarios. |

## 5. Required Learner Materials

Include only items actually used in the module.

| Resource Type | Description |
|---|---|
| Reading Material | LMS lesson text, handouts, reference tables |
| Practice Material | Sample prescriptions, worksheets, scenarios |
| Assessment Tools | Quiz, case questions, rubric, competency checklist |
| Job Aid | Prescription-reading checklist or workflow guide |
| Optional Resources | Review worksheet, glossary, flashcards |

## 6. Module Content Structure

Consistent instructional sequence:

- **A. Introduction to the Topic** — briefly explain the importance of the module topic.
- **B. Key Terms and Definitions** — terminology learners must know before progressing.
- **C. Core Instructional Content** — break into short lessons/units:

  | Lesson Section | Content Purpose |
  |---|---|
  | Lesson 1: Foundational Concepts | Introduce the topic and basic principles |
  | Lesson 2: Required Information | Explain what the learner must identify |
  | Lesson 3: Common Challenges | Present unclear, incomplete, or error-prone situations |
  | Lesson 4: Technician Role | Clarify what the technician should do within the workflow |
  | Lesson 5: Practice Application | Apply learning to examples or scenarios |

- **D. Tables, Checklists, and Job Aids** — preserve where they improve clarity.
- **E. Worked Examples or Scenarios** — case-based, only when supported by source material.
- **F. Knowledge Check** — short embedded questions throughout the module.

## 7. Learning Activities by Modality

Pharmacy technician standards identify **didactic, simulation, and experiential**
modalities, with minimum hour expectations across the full program.

| Modality | LMS Module Use |
|---|---|
| Didactic | Readings, videos, narrated lessons, glossary, knowledge checks |
| Simulation | Practice prescriptions, error-spotting exercises, workflow scenarios, virtual labs |
| Experiential | Observation checklist, site-based practice, preceptor evaluation, competency validation |

For online LMS modules, clearly label whether the activity is didactic instruction,
simulation practice, or experiential preparation.

## 8. Assessment Plan

Every module includes both formative and summative assessment.

| Assessment Type | Purpose | Examples |
|---|---|---|
| Formative | Checks learning during the module | Knowledge checks, drag-and-drop, matching, short scenarios |
| Summative | Confirms achievement of objectives | Final quiz, case analysis, competency checklist |
| Skills/Competency | Confirms practical application | Simulation rubric, prescription review checklist, preceptor validation |

The 2026 ASHP/ACPE standard requires ongoing formative and summative assessment of
student competencies and criteria-based feedback across didactic, simulation, and
experiential modalities.

## 9. Passing Criteria

| Requirement | Standard |
|---|---|
| Module completion | 100% of required lessons completed |
| Quiz score | Minimum passing score set by program policy |
| Attempts allowed | Define number of attempts |
| Remediation | Required review and reassessment if unsuccessful |
| Competency validation | Required where module includes skill demonstration |

## 10. Remediation Plan

1. Learner receives automated feedback or instructor feedback.
2. Learner reviews missed content.
3. Learner completes assigned remediation activity.
4. Learner retakes assessment or completes alternate assessment.
5. Remediation outcome is documented in the LMS.

## 11. Documentation and LMS Evidence

| Evidence Item | LMS Record |
|---|---|
| Completion status | Module completion report |
| Time spent | Seat-time or activity log |
| Quiz score | Gradebook record |
| Assignment submission | Uploaded learner work |
| Rubric score | Instructor or preceptor evaluation |
| Remediation | Retake record or remediation note |
| Version history | Module update log |

## 12. Quality Assurance and Review

| Review Area | Reviewer |
|---|---|
| Content accuracy | Pharmacist SME |
| Instructional design | LMS/instructional designer |
| Assessment alignment | Faculty or program director |
| Accreditation alignment | Accreditation reviewer |
| Grammar and formatting | Editor/QA reviewer |
| Annual update | Program director or designee |

Emphasize continuous quality improvement and maintenance of assessment data for
program improvement.

15. Remediation Instructions

17. References / Source Materials Used
18. Instructor Notes
19. LMS Evidence and Version Control
20. Missing Content / SME Review Notes

## Confirm Before Accreditation Submission

| Item | Why It Matters |
|---|---|
| Accrediting body | Requirements differ by country, institution, and accreditor |
| Jurisdiction | Pharmacy law, technician scope, and practice requirements vary |
| Program hour structure | Must align with total didactic, simulation, and experiential requirements |
| Assessment policy | Passing score, retakes, remediation, and grading must be documented |
| Experiential requirements | Sites, preceptors, supervision, and competency validation must be defined |
| Source material | Modules should not add unsupported laws, standards, or procedures |
| Curriculum map | Every module should map to program outcomes and accreditation standards |

## How this maps to the Pixopharm LMS build (working notes — 2026-06-13)

- **Lesson content blocks** (`lessons.content` JSONB) already supports heading / text /
  callout / key-term / table / video-placeholder / list / divider — covers §6 Core
  Content, §6D tables/job-aids, §6B key terms.
- **Quizzes** (`quiz_questions` + `quiz_cases`) support 8 interactive types — map to
  §6F Knowledge Checks (formative) and §8 Summative. Drag-and-drop/matching/scenario
  cover the §8 formative examples directly.
- **GAPS to design against this format** (not yet built):
  - Module metadata (§1): code/number, seat time, version/review date, author/reviewer.
  - Accreditation crosswalk (§2) and Learning Objectives (§4) as structured fields.
  - Passing criteria / attempts / remediation (§9–10).
  - Competency checklist + experiential evidence (§8 skills, §11).
- **Client will provide a worked sample module** to use as the concrete reference.
- **Requested enhance behavior:** admin picks "create THESE quiz types for this module,"
  AI generates them, admin can then review/modify before saving. (EnhanceDialog already
  has the type-picker; QuizEditor already allows per-type hand editing — flow needs to
  connect generate → review → edit → save against this format.)
